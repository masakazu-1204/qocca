## 🎯 採用方針: B型 (バズ最優先 / X 特化 / 実用情報の密度)

### 媒体ごとの役割分担 (重要)

| 媒体 | 役割 | 文体 |
|---|---|---|
| **X (本書対象)** | バズ最優先 / 新規リーチ / 拡散 | **B型**: 実用密度 + 問い + 具体性 |
| Threads | 既存ファン醸成 / 世界観 / 余韻 | 詩的・編集的トーン (絵文字多め) |
| Instagram | ビジュアル中心 / 静的アーカイブ | Kinfolk 系 (Threads と近い) |
| Blog | SEO / AI 検索引用基盤 | 中立・情報的 (philosophy/general) |

→ **X 版だけ意図的に文体を 振り切る**。詩的・余韻系の締めは Threads/Instagram に譲る。

## 🚫 禁止事項 (B型 厳守 / 詩的トーン 廃止)

X 自動投稿テンプレ 30本 全てで **下記表現を使わない**:

| 禁止 | 理由 |
|---|---|
| 「街の体温」「街の温度」 | 抽象的・詩的 (Threads 寄り) |
| 「静かにそばにある」「ささやかな」 | 余韻系 |
| 「住める速度」「丁寧に」「ぽつり」 | Editorial トーン (本サイト用) |
| 「ぼけぼけ顔」等の Kinfolk 系修辞 | 詩的 |
| 「眺めて元気もらいたい」「励まされる」等の情緒的締め | 拡散しづらい |
| 句読点を 詩的に変える 表現 | 視認性悪化 |
| 改行が 詩的すぎる レイアウト | X タイムラインで弱い |

→ **代わりに採用すべき**: 具体数値 / Step形式 / 「~したい人へ」「~の方は」/ 質問形 / 比較

## ✅ B型 採用原則 (5 つ)

### 1. **実用情報の密度** (最重要)
- 「分かりやすい・覚えやすい・後で見返したい」を最優先
- 抽象論より **具体スペック** (温度 / 日数 / 個数 / 円 等)
- 番号付きリスト or 箇条書きでスキャナビリティ向上

### 2. **リプしたくなる問い**
- 「あなたの場合は?」「教えて」「リプで貼って」等の明示
- 答えやすい簡単な質問 (= 4択 / 5択)
- 「みんなの XX 見てみたい」で UGC 募集

### 3. **保存したくなる具体性** (= ブックマーク獲得)
- 季節情報 (「梅雨対策」「夏の暑さ」「冬の乾燥」)
- 数字付き Tips (「3つのコツ」「5つのサイン」)
- チェックリスト形式

### 4. **ハッシュタグ 1-2 個** (情報タグ重視)
- `#Qocca` は固定
- もう 1 つは **検索性のあるタグ** (`#犬のいる暮らし` / `#猫のいる暮らし` / `#暑さ対策` 等)
- バイラル狙いの過剰タグ (`#拡散希望` 等) は禁止

### 5. **絵文字 1-2 個** (節度)
- 行頭の絵文字 (アイキャッチ) 1 個
- 文中の絵文字 1 個まで (例: 🐾)
- 装飾としての絵文字連発禁止 (Threads と差別化)

## 📋 投稿タイプ 5 種類 (B型 4本柱対応)

### タイプ1: `tip_practical` (実用Tips / 8本)

**目的**: ブックマーク獲得 (4本柱 #3)

**文体**:
- 番号付きリスト or 箇条書き必須
- 季節 / 安全 / お手入れ系
- **医療系の断定 NG** (健康記録 設計憲法と同じ)
  - ❌ 「~は危険」「~するべき」 (断定)
  - ✅ 「~は要注意」「~の方は獣医師にご相談を」 (余地)
- 出典必要な数値は使わない or 一次情報 (AVMA / 厚労省 等) のみ

**画像**: 該当 Tips のシーン (犬が水を飲む / 猫がブラッシングされる 等)

**プロンプト枠組み**:
```
🌞 {季節or状況}に必要な、{ペット}の{ジャンル}を{N}つ

① {Tip 1}
② {Tip 2}
③ {Tip 3}

{ベネフィット 1文}
気になる症状は獣医師にご相談を。

#Qocca #{季節or対象タグ}
```

### タイプ2: `question_engage` (問いかけ / 7本)

- ハッシュタグはオリジナル化 OK (`#うちの子起こし方` 等)

**画像**: 問いの主題に沿うシーン

### タイプ3: `showcase_pet` (うちの子募集 / 6本)

**目的**: UGC 写真投稿の誘発 (4本柱 #2)

- 「ぜんぶ見ます」等のリアクション約束
- 季節 / 曜日 / 状況をフックに

**画像**: 例示シーン (具体ペット写真風)

**プロンプト枠組み**:
```

✓ {観点1}
✓ {観点2}
✓ {観点3}

写真と一緒に リプで貼ってくれたら ぜんぶ見ます。

#{オリジナルタグ}
```

### タイプ4: `featured_creator` (出品者/作品紹介 / 5本)

**目的**: 出品者誘導 + 制作プロセス価値訴求 (4本柱 #1 + #2)

**文体強化** (Phase B 修正):
- ❌ 感傷的引用 / 「静かな声」「あの子の」等の Threads 寄り表現は **廃止**
- ✅ **制作工程 / 素材 / 所要日数 / 価格レンジ** 等の **具体スペック** を前面
- ✅ 「他と何が違うか」を1文で
- ✅ 詳細は qocca.pet で見られる旨を末尾に

**画像**: 制作風景 / 完成品 / 素材

**プロンプト枠組み**:

### タイプ5: `behind_scene` (Qocca 裏側 / 4本)

**目的**: 開発進捗の透明性 + 機能訴求 (4本柱 #2)

## 🔄 ロールバック手順 (緊急時)

新テンプレが想定外の挙動を見せた場合、以下 SQL で即時 ロールバック可能:

```sql
-- A. 即時 緊急停止 (Kill Switch ON)
UPDATE x_post_settings SET kill_switch = true WHERE id = 1;
-- → cron は走るが post 直前で 503 返してスキップ (Meta テストイベント風)

-- B. 旧 46本に戻す (新30本を無効化 + 旧46本を有効化)
UPDATE x_post_templates SET is_active = false
  WHERE theme IN ('tip_practical', 'question_engage', 'showcase_pet', 'featured_creator', 'behind_scene');
UPDATE x_post_templates SET is_active = true
  WHERE theme IN ('ARK 動物福祉', 'KPI', 'Qocca 裏側', 'うちの子の物語', 'クリエイター作品紹介',
                  'クリエイター紹介', 'ビジョン', 'ペット業界考察', '一週間振り返り',
                  '動物福祉', '哲学', '施設・イベント', '明日への準備', '開発者の声');

-- C. 設定も戻す (画像必須化を元に戻す)
UPDATE x_post_settings SET morning_use_image = false WHERE id = 1;
```

→ Kill Switch なら **5 秒で安全停止**、ロールバックなら **10 秒で旧運用に復帰**。
